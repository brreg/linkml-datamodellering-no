# ./specs

Her dokumenterer vi større endringar i repoet ved hjelp av endringsplanar genererte av KI.

## Struktur

- `backlog/` — aktive specs: framtidig arbeid, opne forslag, pågåande vurderingar
- `done/` — avslutta specs: implementerte features og gjennomførte analysar
- `rejected/` — avviste specs: forslag som er forkasta med grunngjeving
- `bugs/` — kjente feil med aktive workarounds

Nye specs opprettast alltid i `backlog/`. Når arbeidet er avslutta, flyttast fila til `done/`. Forslag som vert forkasta, flyttast til `rejected/` med ein `## Avvist`-seksjon øvst som forklarer kvifor.
